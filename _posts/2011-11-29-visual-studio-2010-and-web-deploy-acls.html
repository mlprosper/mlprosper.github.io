Recently I’ve started using Web Deploy directly from Visual Studio 2010.  I had everything working on the staging server at one point and stupidly didn’t record the folder specific ACL changes I had made to allow IIS to serve the assets (css, images) correctly.  The first time I used Web Deploy, all the ACLs were overwritten, and my website was no longer accessible!  After some research, I found an article detailing how to prevent VS 2010 from setting the ACLs on the target automatically:

http://www.orcsweb.com/blog/rick/prevent-visual-studio-2010-and-web-deploy-from-changing-acls/
