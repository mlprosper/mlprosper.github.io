I needed to set up a new load balancer in Amazon Web Services because my previous SSL cert had expired.  The AWS management console doesn’t allow you to enter a new SSL cert apart from setting up a new ELB.  After a little research, I found a step by step tutorial laying out how to generate a new CSR and install the certificate:

http://stackoverflow.com/questions/6753619/how-do-i-setup-ssl-certs-on-amazon-aws-load-balancer-using-godaddy-certificates

I have Git for Windows installed and it comes with openssl.  You can access the openssl executable here:

c:\Program Files (x86)\git\bin

When you run the commands, for some reason the files are dumped in a strange folder:

C:\Users\Mike\AppData\Local\VirtualStore\Program Files (x86)\Git\bin

A quick search will turn up the path where those files were dumped.
